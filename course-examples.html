Highlighting

Highlighting is a search-parameter:

{
 "query" : {
  "match" : {
   "description" : "woodstock"
  }
 },
 "highlight" : {
  "pre_tags" : [ "<b>" ],
  "post_tags" : [ "</b>" ],
  "fields" : {
   "description" : {"fragment_size" : 100, "number_of_fragments" : 5}
  }
 }
}

More settings in the documentation:
Search APIs » Request Body Search » Highlighting

Highlighting in Scala/Play

Make a highlighting query for "woodstock".

MultiMatchQuery(fields = Seq("name", "description"), value = "woodstock").
  withHighlight(HighlightField("name"), HighlightField("description"))
MultiMatchQuery(fields = Seq("name", "description"), value = "woodstock").
  withHighlight(
    Highlight(fields = Seq(HighlightField("name"), HighlightField("description")),
              tags = Some(Seq(("""<span class="hilite">""", "</span>"))))
  )

Facets

Facet results for a type with a 'country' field could look like: 


"facets" : {
  "country_facet" : {
    "terms" : [
      { "term" : "Nepal", "count" : 11 },
      { "term" : "Netherlands", "count" : 37 },
      ... 
    ]
  }
}

Facets using query

Returns only one field-value in the facets. 


{
  "query" : { ... },
  "facets" : {
    "dutch" : {
      "query" : {
        "term" : { "country" : "netherlands" }
      }
    }
  }
}

Facets using filter

Almost the same as query, but faster. 


{
  "query" : { ... },
  "facets" : {
    "dutch" : {
      "filter" : {
        "term" : { "country" : "netherlands" }
      }
    }
  }
}

Facets using terms

Returns the most common terms in a field. 


{
  "query" : { ... },
  "facets" : {
    "country_facet" : {
      "terms" : {
        "field" : "country",
        "size": 10, // 10 most common terms
        "order" : "count", // term, reverse_count, reverse_term
        "all_terms" : false // true: even those with zero count
      }
    }
  }
}
More parameters in [Search APIs » Facets » Terms Facet].

Autocomplete / suggestions

Using the prefix query


{ "query" :
  {
    "prefix" : { "country": "ne" } 
  } 
}

This has performance problems, because it iterates through all terms when executing the query.

Autocomplete using edgengram

Split terms during indexing, so "Netherlands" becomes:
["Ne", "Net", "Neth", "Nethe", "Nether", "Netherl", "Netherla", "Netherlan"]

Index analysis: 


"filter" : {
  "autocomplete" : {
    "type" : "edgeNGram",
    "min_gram" : 2, "max_gram" : 9
  }
}
Query: 

{ "query" :
  {
    "term" : { "country": "ne" } 
  } 
}

Autocomplete with multi_field

Mapping: 


{
  "country" : {
    "type" : "string", "index" : "analyzed",
    "fields" : {
      "label" : { "type" : "string", "index" : "not_analyzed" },
      "autocomplete" : {
        "type" : "string", "index" : "analyzed",
        "index_analyzer" : "autocomplete", "search_analyzer" : "standard"
      }
    }
  }
}
Query: 

{
  "size" : 0,
  "query" : { "term" : { "country.autocomplete" : "ne" } },
  "facets" : {
    "country" : { "terms" : { "field" : "country.label" } }
  }
}
